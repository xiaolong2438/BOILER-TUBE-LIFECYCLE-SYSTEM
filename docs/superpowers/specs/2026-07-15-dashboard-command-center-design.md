# 锅炉炉管生产指挥中心主页设计

## 目标

将现有主页改造成适合设备专工日常使用的三栏生产指挥中心。借鉴参考图的中心聚焦和左右信息分区，但不照搬高密度装饰边框、虚构仪表或营销式文案。

## 布局

桌面端首屏采用 26% / 48% / 26% 三栏结构：

- 左栏：严重风险、预警数量、今日处置任务和快捷操作。
- 中栏：当前机组、综合健康度核心环图、风险等级分布和历次检修健康趋势。
- 右栏：各受热面健康度排名、最近预警和数据更新时间。
- 底部状态带：生命周期闭环率、台账覆盖、检测完整率和下次检修节点。

首屏不使用解释性口号。页面标题、机组切换、状态名称和必要数据标签之外，不增加可见的功能说明文字。

## 核心组件

### 左栏

- 四个紧凑指标：严重风险、预警关注、管理单元、闭环率。
- 三条处置任务，按 P1、P2、P3 排序并保留“打开”动作。
- AI 风险摘要、管段综合分析、检修计划三个入口。

### 中栏

- 全厂、1#、2# 机组切换位于中栏顶部。
- 中央环图展示综合健康度，外圈按正常、关注、预警、严重分层。
- 环图两侧展示严重风险数量、预警数量、较上次检修变化和当前闭环率。
- 下方折线图以历次检修节点为横轴，不使用月度趋势。

### 右栏

- 各受热面健康度按从低到高排序，保留点击进入部件档案。
- 最新预警最多显示五条，避免与左栏处置任务重复描述。
- 展示当前机组、数据更新时间和数据来源状态。

## 数据与交互

- 继续复用现有 `dashboardUnit`、`collectDashboardWarnings`、`getDashboardTrendStats` 和受热面健康度数据。
- 机组切换同时刷新环图、检修趋势、受热面排名、预警和任务可见性。
- 环图、排名和预警均可进入现有管段或部件详情，不新增独立业务页面。
- 数据为空时显示简短空状态，不生成虚构预警。

## 视觉语言

- 使用深色工业背景和青色结构线，红、黄、绿只表达风险状态。
- 中央环图是唯一强视觉中心；左右栏使用低对比面板和细边框。
- 不使用装饰性数字翻牌、无业务含义的小仪表、动画网格或过度发光。
- 卡片圆角不超过 8px，文字尺寸服从信息密度，不使用大标题占据工作区。

## 响应式

- 1100px 以下改为中栏、左栏、右栏的纵向顺序。
- 700px 以下指标改为两列，处置任务和排名改为单列，所有按钮允许换行。
- 环图保持稳定宽高比，文字和指标不得遮挡图形。

## 验证

- 验证全厂、1#、2# 切换后数据同步更新。
- 验证所有“打开”和快捷入口仍可用。
- 验证桌面和移动视口无重叠、截断和空白大块。
- 运行页面脚本语法检查，并在本地浏览器检查首屏布局。
